My Inventory

A simple inventory system for Unity3d

Asset Store • Feedback

Creating an inventory system that works for your project can be time-consuming for new or even experienced game developers. My Inventory aims to be easy to implement, and to seamlessly complement your Unity game no matter what it is.





Getting Started

Purchasing My Inventory from the Unity Asset Store gives you access to all the source files and code that builds the My Inventory system. You can edit or add new features to the source code to fit your project, but My Inventory is NOT open source.

You can read the entire software license by downloading the LICENSE file.

The My Inventory Unity asset comes with demo files and scenes displaying a complete inventory setup to make it easy for you to get started right ahead, but for a complete set-up guide you can continue reading!

Creating an InventorySkin

Before anything, you need to decide how your inventory should look, and to do that you need to create and configure an InventorySkin.

To create an InventorySkin, right-click in your asset browser,
Create > Inventory > Inventory Skin

In your InventorySkin you can add sprites for the different elements of your inventory. If you want all the different sprites of your inventory to have the same pixel size, all of your sprite assets should have the same 'Pixels Per Unit' value.

When you have added your sprites and optionally a font to your InventorySkin through the inspector, it helps to have your inventory display on the screen before tweaking the other skin settings. An InventorySkin can be modified at runtime, and we will take advantage of this to easily see our inventory get updated as we tweak the InventorySkin values.

Please note that not all of the InventorySkin variables support runtime editing. You might occasionally need to restart your player to see changes, but to see changes in the first place you will need to render your inventory to the canvas using an InventoryImage.

Creating an InventoryImage

To display your inventory to the screen, you need to create an InventoryImage object by right-clicking in the scene hirearchy,
UI > Inventory Image
and place it on a Canvas.

You can add your InventorySkin to the editorSkin property of your InventoryImage in the inspector to see an approximation of your inventory on the camera in edit mode.
It is possible to scale the InventoryImage on the x and y axis and move it around the screen to position it.

InventoryImage in edit mode

Now it's time to write some code to load and draw your inventory.
In a new script, add the following code:

public class MyScript : InputHandler {

    Inventory inventory;

    public InventorySkin skin;
    public InventoryImage image;

    private void Awake() {
        inventory = Inventory.Load("inventory", skin);
    }

    void Update() {
        inventory.Draw(image);
    }

}

Add the script to an object and then add your InventorySkin and your newly created InventoryImage to the skin/ image properties respectively.
When you start the player, you should now see your inventory rendered to the screen. This is the perfect time to tweak your InventorySkin properties to perfection!

Creating an ItemShelf

To add items to your inventory, you need an ItemShelf present in a /Resources/ folder somewhere in your project. When you first import the My Inventory package to your project, there is a pre-configured ItemShelf in the /My Inventory/Demo/Resources/ folder. You will need to delete the pre-existing one if you want to create your own, as there can only be one ItemShelf in the project at once.

You can edit the Item.cs script to add custom variables to your items to organize and distinguish them further, as the Item.cs class initially only holds a sprite and a name variable.
The Item.cs variables does not need to be serializable, as the Inventory Slots holds the item only as an id.

inventory.PlaceItem(ItemShelf.Random, 0, 0, 1); This will take a random item from the ItemShelf, and place it in slot(0, 0) with a count of 1.
